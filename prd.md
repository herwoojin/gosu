# 우리동네고수 — PRD.md (Product Requirements Document)

> 본 문서는 **무엇을(What)** 만들지를 정의한다. 구현 방법(How)은 TRD에서 다룬다.
> 우선순위: **M(Must) / S(Should) / C(Could) / W(Won't-now)**

---

## 1. 페르소나

### P1. 경영주(사용자, OWNER) — "김점주"
- 전국 편의점 점포를 운영. 냉난방 고장·누수·철거 등 발생 시 **빠르게** 해결하고 싶다.
- IT 비전문가, 모바일로만 사용. 복잡한 절차 싫어함. "여러 업체 가격을 한 번에 비교"가 핵심 니즈.

### P2. 협력사(고수, PARTNER) — "이고수"
- 냉난방/설비/철거 등 1~다수 공종 보유. 개인사업자 또는 무사업자 숙련공일 수 있음.
- 내 지역·내 공종에 맞는 일감 알림을 받고 입찰하고 싶다. 정산이 투명·신속하길 바람.

### P3. 본부 시설담당자(관리자, ADMIN) — "박과장"
- 전국 점포의 시설 업무를 총괄. 협력사 직접 수소문에 지친 상태.
- 협력사 검증·평가, 본부결제 승인, 단가 비교, 데이터 자산화가 핵심 업무.

### P4. 슈퍼관리자(SUPER_ADMIN)
- 관리자 지정 권한, 공종/지역/단가 마스터, 시스템 설정.

## 3. 핵심 기능 요구사항

### 3.1 인증/계정 (M)
- Firebase **Google 로그인 + 이메일/비밀번호** 로그인.
- 최초 로그인 시 **역할 선택/배정**: OWNER는 셀프 가입, PARTNER는 가입 후 **승인 대기**, ADMIN은 지정만 가능.
- 이메일 인증, 비밀번호 재설정, 휴면/탈퇴.
- 계정 1개당 다중 역할 가능(예: 경영주이면서 협력사) — 단 역할 전환은 명시적.

### 3.2 견적요청 마법사 (3~5단계, 숨고 유사) (M)
공종별로 질문 스키마(JSON)를 다르게 구성. 기본 골격:
1. **공종 선택** (냉난방기 수리/청소, 철거·원상복구, 폐기물 처리, 수도배관, 누수탐지, 용달·이사, 테라스(방부목), 무촌철거 등)
2. **세부 서비스 선택** (예: 냉난방기 → 설치/수리/점검/이전설치)
3. **매장·위치 정보** (주소→좌표 변환, 점포 유형, 면적 등)
4. **일정·긴급도** (즉시/오늘/이번주/협의, 긴급 가산)
5. **상세 요청 + 사진/동영상 첨부** (선택)

- 단계 수는 공종에 따라 3~5로 가변(`request_templates`로 관리, ADMIN 편집).
- 임시저장/이어쓰기, 진행률 표시, 모바일 큰 터치 영역.

### 3.3 매칭 엔진 (M)
- 입력: 요청의 **공종 + 위치(좌표) + 긴급도**.
- 후보 산출: 해당 **공종 보유** AND 해당 **서비스지역(반경/시군구) 포함** 협력사.
- 정렬 가중치(기본): `0.5·평점 + 0.3·근접도 + 0.2·응답성/낙찰이력` (관리자 조정 가능).
- **잘하는 협력사 우선 노출**: 공종×지역별 평점이 매칭·노출 순위에 반영.
- 매칭 결과를 후보 협력사에게 알림 발송.

### 3.4 입찰(역경매) (M)
- 협력사: 관심 건에 **입찰가 + 메시지 + (선택)견적서 첨부** 제출. 마감시간/최대입찰수 설정.
- 경영주: 입찰 목록을 **가격·평점·후기·예상일정**으로 비교 → 낙찰.
- 낙찰 시 양측 연락처·상세 공개(통신판매중개 고지 동반).
- 입찰 취소/유찰/재요청 처리.

### 3.5 협력사 등록·검증 (M)
- 등록 트랙 3종:
  - **법인/개인사업자**: 사업자등록번호 입력 → **국세청 진위·상태 확인** 통과 시 등록.
  - **무사업자 개인(숙련공)**: 사업자 없이 신원·자격·포트폴리오·평가 기반 등록(원천징수 안내).
- 공통 입력: 상호/이름, 대표공종(복수), **서비스 지역**(주소→시군구/반경), 연락처, 자격증·보험·포트폴리오 업로드.
- ADMIN **승인/반려** 워크플로우(반려 사유 기록).
- **거짓 경력 방지**: 진위확인 + 자격증 검증 + 후기/평점 + 분쟁이력 노출(완전 차단은 불가하나 신뢰 신호 다층화).

### 3.6 평가 시스템 (이원화) (M)
- **사용자평가(OWNER)**: 별점 + 항목별(품질/가격합리성/친절/시간준수) + 후기 텍스트/사진.
- **관리자평가(ADMIN)**: 계약준수/정산/클레임대응/안전/서류완비 등.
- 합산 점수 = 가중 평균(공개용 종합점수 + 내부용 관리자점수 분리).
- 공종×지역별 점수 별도 산출 → 매칭 노출에 반영.
- **이의신청/정정 절차**(명예훼손 방지), 거짓후기 신고.

### 3.7 결제·정산 (S→Phase2)
- **즉시결제**: 경영주가 PG(토스페이먼츠 등)로 결제, 에스크로 → 작업완료 확인 후 협력사 정산.
- **본부결제**: 경영주가 "본부결제" 선택 → 본부 시설담당자에게 매칭 큐로 상신 → **ADMIN 승인** → 비용은 **본부에서 협력사로 직접 지급**(플랫폼은 자금 미보관, 승인·기록만).
- 세금계산서/현금영수증/지급명세서 연동, 무사업자 **3.3% 원천징수** 안내.

### 3.8 알림 (M)
- FCM 웹푸시(PWA): 신규 매칭/입찰/낙찰/정산/평가 요청.
- (옵션) 카카오 알림톡(협력사 대상).
- 수신동의 관리 + 야간 전송 제한.

### 3.9 단가 데이터 & 대시보드 (S→Phase2)
- 입찰·낙찰가 누적 → **공종별 전국단가 / 지방단가 / 현재단가** 비교 차트.
- 매년 시세 갱신, 표준인건비·유류대(오피넷) 참고치 병행 표시("참고용" 고지).
- ADMIN 전용 분석 화면.

### 3.10 AI 견적서 생성 (S→Phase2)
- 견적서 양식이 없는 협력사를 위해 **Claude API로 표준 견적서 양식 자동 생성**(공종·내역 기반).
- 사용자는 항목/단가 입력만으로 PDF 견적서 산출. **거짓·과장 금지 가이드** 포함.

### 3.11 PWA (M)
- manifest + Service Worker(오프라인 캐시·설치 프롬프트).
- 모바일/태블릿 앱처럼 동작 → **PWABuilder로 APK** 생성/배포.

## 4. 대표 유저스토리

- **U1.** 경영주로서, 냉난방기가 고장나면 3~5단계만 입력하고 여러 업체 입찰을 받아 가격·평점으로 비교하고 싶다.
- **U2.** 협력사로서, 내 공종·지역에 맞는 일감 알림을 받고 즉시 입찰하고 싶다.
- **U3.** 협력사로서, 견적서 양식이 없어도 AI가 만들어준 양식으로 깔끔히 제출하고 싶다.
- **U4.** 무사업자 숙련공으로서, 사업자등록 없이도 검증·평가로 일감을 받고 싶다.
- **U5.** 본부 담당자로서, 협력사를 직접 찾지 않고 매칭 큐에서 승인·정산만 하고 싶다.
- **U6.** 본부 담당자로서, 공종별 전국/지방/현재 단가를 한눈에 비교하고 싶다.
- **U7.** 슈퍼관리자로서, 신뢰할 사용자를 관리자(ADMIN)로 지정하고 싶다.
- **U8.** 경영주로서, 결제를 즉시 하거나 본부결제로 돌릴지 선택하고 싶다.

## 5. 비기능 요구사항 (NFR)

- **모바일 우선**: 1열 카드, 큰 버튼(최소 44px), 한손 조작.
- **접근성**: 고령 경영주·시각 약자 고려(큰 글씨 옵션, 명도 대비, 음성입력 호환).
- **성능**: 첫 화면 LCP < 2.5s(3G 기준 합리적), 매칭 응답 < 2s.
- **보안**: RLS로 역할별 데이터 격리, 민감정보(주민번호) 최소수집·암호화.
- **국문 UI** 기본, 어투는 친절·간결.
- **감사로그**: 권한변경·승인·결제·평가 핵심 이벤트 기록.

## 6. 화면 목록(IA, 모바일)

- 공통: 스플래시 / 로그인(구글·이메일) / 역할선택 / 홈
- OWNER: 홈(주요 공종 그리드) → 견적요청 마법사 → 입찰현황 → 입찰비교/낙찰 → 결제선택 → 평가
- PARTNER: 홈(매칭 일감) → 일감상세 → 입찰 → 내 입찰/낙찰 → 정산 → 프로필/공종/지역 관리
- ADMIN: 대시보드 → 협력사 승인 → 본부결제 큐 → 평가관리 → 단가 분석 → 분쟁
- SUPER_ADMIN: 관리자 지정 → 마스터(공종/지역/단가/템플릿) → 시스템설정
