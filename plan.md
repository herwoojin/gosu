# 우리동네고수 (Our Neighborhood Master) — PLAN.md

> 전국 편의점 운영 본부 시설지원팀을 위한 **입찰형(역경매) 협력사 매칭 모바일 웹앱**
> 본 문서는 기획 의도를 정리한 최상위 계획서이며, 후속 문서(PRD/TRD/ERD/GUIDE/TASK)의 기준이 된다.

---

## 0. 한 줄 정의

**"숨고(Soomgo)의 입찰 방식 + 기업 본부의 시설관리 워크플로우 + 전국 협력사 데이터 자산화"를 하나로 묶은 원스톱 유지보수 매칭 플랫폼.**

- 요청자(경영주)는 내 점포 근처의 **검증된 좋은 협력사**를 빠르게 입찰로 구한다.
- 협력사는 **공종·지역·평점 기반 매칭**으로 자기에게 맞는 일감 알림을 받고 입찰한다.
- 본부 시설담당자는 직접 업체를 검색하지 않고, **매칭 툴 + 누적 단가 데이터**로 전국 업무를 한 번에 처리한다.

---

## 1. 배경과 문제 정의

| 현재의 문제 | 우리동네고수가 해결하는 방식 |
|---|---|
| 시설담당자가 매번 협력사를 직접 수소문 → 시간 소모 | 공종·지역·평점 기반 자동 매칭 + 입찰 |
| 경영주가 고장·누수·철거 시 어디에 맡길지 모름 | 3~5단계 질문 후 다수 협력사 입찰 비교 |
| 협력사의 실력·신뢰도를 알 수 없음 | 사업자진위확인 + 사용자평가 + 관리자평가 이원화 |
| 단가가 지역·시기마다 제각각, 비교 불가 | 전국단가·지방단가·현재단가 누적 비교 |
| 좋은 협력사 정보가 담당자 개인에게만 있음 | 본부 차원의 협력사 데이터 자산화(매년 시세 갱신) |
| 결제·정산 신뢰 문제 | 즉시결제 / 본부결제 2-트랙 + 통신판매중개 구조 |
| 사업자 없는 숙련공은 진입 불가 | 무사업자도 검증·평가 기반으로 참여 가능 |

---

## 2. 핵심 가치 제안 (Value Proposition)

1. **경영주(요청자):** 시간 손실 없이, 점포 주변 좋은 협력사를 신속·투명하게 입찰로 매칭.
2. **협력사(고수):** 초보부터 시작해도 검증·평가로 성장. 거짓 없는 경력이면 누구나 일감.
3. **본부 시설팀:** 전국 업무를 한 화면에서 처리 + 협력사/단가 데이터 자산화 + 표준인건비·유류대 실시간 비교.
4. **공통:** "최저가이면서 고품질" — 가격과 평점을 동시에 비교하는 의사결정.

## 3. 3개의 사용자 모드 (로그인 시 분기)

| 모드 | 대상 | 핵심 권한 |
|---|---|---|
| **관리자 모드 (ADMIN)** | 본부 시설담당자/운영자 | 협력사 승인·평가, 사용자 중 관리자 지정, 본부결제 승인, 단가/공종/지역 마스터 관리, 분쟁 처리 |
| **사용자 모드 (OWNER)** | 전국 편의점 경영주(점포 운영자) | 견적요청 생성(3~5단계), 입찰 비교·선택, 즉시결제/본부결제 선택, 사용자평가 작성 |
| **협력사 모드 (PARTNER)** | 유지보수 업체 / 개인사업자 / 무사업자 숙련공 | 프로필·공종·지역 등록, 사업자진위확인, 알림 수신, 입찰, 견적서 제출, 정산 |

> 관리자 권한은 **지정된 관리자만** 부여 가능(자가 승격 불가). 기존 관리자가 OWNER 사용자 중에서 ADMIN을 지정.

## 4. 대표 플로우 (개념)

```
[경영주] 견적요청 생성(공종→세부→매장/위치→일정/긴급도→상세/사진)
        │
        ▼
[매칭엔진] 공종 + 지역(PostGIS 반경) + 평점 가중치로 후보 협력사 산출
        │
        ▼ (FCM 웹푸시 / 알림톡)
[협력사] 관심 건에 입찰(가격·메시지·견적서)  ← 견적서 없으면 AI가 양식 생성
        │
        ▼
[경영주] 입찰 비교(가격·평점·후기) → 선택
        │
        ├── 즉시결제 (PG: 토스페이먼츠 등) ─────► 작업 → 사용자평가
        └── 본부결제 (본부 시설담당자 승인) ─────► 본부→협력사 직접 정산 → 사용자평가 + 관리자평가
```

## 5. 범위(Scope) — 단계별

### MVP (Phase 1)
- 3개 모드 로그인(Firebase Google + Email) 및 역할 분기
- 협력사 등록 + 사업자진위확인(국세청 공공데이터 API)
- 견적요청 3~5단계 마법사 + 입찰
- 매칭(공종+지역+평점) + 알림(FCM)
- 사용자평가 / 관리자평가
- PWA(설치형) + Paperlogy 폰트

### Phase 2
- 즉시결제(PG) + 본부결제 승인 워크플로우 + 정산
- 단가 데이터(전국/지방/현재) 비교 대시보드
- AI 견적서 양식 자동 생성
- 무사업자/개인사업자 참여 트랙 + 원천징수 안내

### Phase 3
- 표준인건비·유류대(오피넷) 실시간 비교
- 협력사 등급/뱃지/성장 트래킹
- 분쟁·클레임 관리, 정산 자동화 고도화
- PWABuilder APK 배포

## 6. 법적 컴플라이언스 개요 (상세는 PRD/TRD에)

> ⚠️ 아래는 일반적 가이드이며, 실제 출시 전 **변호사·세무사 검토 필수**. 본 문서는 법률 자문이 아니다.

1. **통신판매중개업자 지위 (전자상거래법 제20조)**
   - 우리동네고수는 거래 당사자가 아니라 **중개자**임을 명확히 고지("거래의 당사자가 아니며 책임은 거래 당사자에게 있음").
   - 협력사(판매자)의 사업자 정보·연락처를 요청자에게 제공.
   - 통신판매중개업 신고 검토.

2. **사업자등록 진위확인 (국세청 / 공공데이터포털 data.go.kr)**
   - "사업자등록정보 진위확인 및 상태조회" API로 사업자등록번호의 **진위 + 상태(계속/휴업/폐업)** 검증 후 등록 승인.

3. **결제·정산 (전자금융거래법)**
   - 플랫폼이 자금을 직접 보관/이체하면 전자금융업(PG/에스크로) 등록 이슈 발생.
   - 따라서 **즉시결제는 라이선스 PG(토스페이먼츠 등) + 에스크로**를 사용하고, 플랫폼은 자금을 직접 커스터디하지 않음.
   - **본부결제는 B2B 정산** — 본부(법인)가 협력사에 직접 지급, 플랫폼은 승인·기록만 담당.

4. **무사업자/개인 참여 (소득세법)**
   - 인적용역 사업소득 **3.3% 원천징수** 또는 기타소득 처리.
   - 지급 주체(본부 또는 PG)가 원천징수 의무. 플랫폼은 정산 명세·지급명세서 작성을 지원.
   - **근로자성/위장도급 위험 방지**: 독립계약(도급) 명시, 지휘·감독 관계 배제.

5. **개인정보보호법 / 정보통신망법**
   - GPS 위치, 사업자정보, (원천징수용) 주민번호는 **법령 근거·별도 동의** 하에 수집.
   - 알림(광고성 정보)은 수신동의 + 야간(21~08시) 전송 제한 준수.

6. **후기·평점 (전자상거래법)**
   - 거짓 후기 금지, 후기 운영원칙 공개, 협력사 이의신청·정정 절차 마련(명예훼손 방지).

7. **단가/표준인건비 정보**
   - 시중노임단가·유가(오피넷 공개데이터) 등 **공개 출처 기반** 참고치로 제공, "참고용" 명시.

## 7. 기술 스택 (요약, 상세는 TRD)

- **인증:** Firebase Authentication (Google OAuth + Email/Password)
- **데이터/매칭:** Supabase (PostgreSQL + **PostGIS** + RLS) — 공종·지역 지오매칭, 평점, 단가
- **인증↔데이터 연동:** Supabase Third-Party Auth로 Firebase JWT 수용(RLS는 Firebase uid 기준)
- **프론트:** Next.js 14(App Router) + TypeScript + Tailwind + shadcn/ui, **모바일 우선**
- **알림:** FCM(웹푸시) + (옵션) 카카오 알림톡
- **결제:** 토스페이먼츠(즉시결제·에스크로) / 본부결제 워크플로우
- **AI:** Anthropic Claude API(견적서 양식 자동 생성, 요청 요약)
- **외부 API:** 국세청 사업자진위확인, 도로명주소(행안부/카카오), 오피넷 유가
- **PWA:** manifest + Service Worker → PWABuilder로 APK 생성
- **폰트:** **Paperlogy(페이퍼로지)** 자체 호스팅(woff2)
- **배포:** Vercel 또는 Netlify

> 사용자는 본 문서를 바탕으로 **Google Antigravity IDE + Claude Opus**에서 재설계·구현 예정.

## 8. 성공 지표 (KPI)

- 견적요청 → 첫 입찰까지 평균 시간(목표: 30분 이내)
- 요청 대비 매칭 성사율(목표: 80%+)
- 협력사 진위확인 통과율 / 재참여율
- 본부결제 비중 및 정산 리드타임
- 공종별 누적 단가 데이터 건수(자산화 정도)
- 평균 평점 및 분쟁 발생률
